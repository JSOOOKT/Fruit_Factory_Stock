## 1. ภาพรวมโครงการ / Project Overview

**ชื่อแอป / App Name:** Fruit Factory Stock

แอปพลิเคชันสำหรับจดบันทึกการรับเข้า–ส่งออก (Stock In / Stock Out) ของวัตถุดิบในโรงงานผลไม้/แช่สาร
โดยอ้างอิงจากสมุดบันทึกกระดาษ/สเปรดชีตเดิมที่ใช้บันทึก "รับเข้ามะละกอฝา แช่สาร" แยกตามประเภทฝา
(ฝาแดง, ฝา 2 สี, ฝาขาวแดง, ฝาเหลือง, ฝาขาวเหลือง ฯลฯ) พร้อมชื่อผู้ส่ง วันที่รับเข้า จำนวนกิโลกรัม
ยอดรวม ยอดที่นำไปผลิต และยอดคงเหลือ

This app digitizes the paper/spreadsheet-based logbook currently used to record incoming raw
material (papaya/fruit, preserved with chemical solution — "แช่สาร") by cap/type category, sender
name, receiving date, weight in KG, running totals, quantity issued to production, and remaining
balance.

### ปัญหาที่ต้องการแก้ / Problem Statement
- พนักงานหน้างานส่วนใหญ่เป็นแรงงานที่ไม่ถนัดพิมพ์/ใช้คอมพิวเตอร์ ต้องการวิธีบันทึกที่เร็วและง่าย (พูดแทนพิมพ์)
- มีพนักงานหลายคนสลับกันจดตามกะรายวัน ต้องรู้ว่า "ใครจด" และ "วันไหน"
- ข้อมูลปัจจุบันอยู่ในสเปรดชีต เสี่ยงพิมพ์ผิด คำนวณผิด และไม่มีประวัติการแก้ไข
- ต้องใช้งานได้ทั้งภาษาไทยและอังกฤษ (โรงงานมีทั้งพนักงานไทยและอาจมีผู้บริหาร/ลูกค้าต่างชาติ)

## 3. กลุ่มผู้ใช้งาน / User Personas

1. **พนักงานจดบันทึก (Recorder / Shift Worker)**
   - ใช้งานหน้างาน มือเปื้อน/ไม่สะดวกพิมพ์ ต้องการปุ่มใหญ่ ใช้เสียงพูดเป็นหลัก
   - ทักษะดิจิทัลต่ำ-ปานกลาง
2. **หัวหน้ากะ / ผู้ควบคุมคลัง (Shift Supervisor / Stock Controller)**
   - ตรวจสอบ แก้ไข อนุมัติรายการ, ดูสรุปยอดประจำวัน
3. **ผู้จัดการ/เจ้าของโรงงาน (Manager / Owner)**
   - ดูรายงานสรุป แนวโน้ม ยอดคงเหลือ ประวัติผู้ส่งแต่ละราย
4. **ผู้ดูแลระบบ (Admin)**
   - จัดการรหัสสินค้า ประเภทสินค้า ผู้ใช้งาน สิทธิ์การเข้าถึง

## 4. ขอบเขตฟีเจอร์ / Feature Scope

### 4.1 การจัดการรหัส/ประเภทสินค้า (Product Code & Type Master Data)
- สร้าง/แก้ไข **รหัสสินค้า (Product Code)** เช่น `RD-001`, `2C-002`
- ระบุ **ชนิด/ประเภทสินค้า (Product Type)** เช่น:
  - ฝาแดง (Red Cap)
  - ฝา 2 สี (Two-Color Cap)
  - ฝาขาวแดง (White-Red Cap)
  - ฝาเหลือง (Yellow Cap)
  - ฝาขาวเหลือง (White-Yellow Cap)
  - เพิ่มประเภทใหม่ได้ไม่จำกัด (extensible list)
- แต่ละประเภทมีหน่วยนับเป็นกิโลกรัม (KG) เป็นค่าเริ่มต้น

### 4.2 บันทึกรับเข้า (Stock In Entry)
ฟิลด์ข้อมูลต่อ 1 รายการ:
| ฟิลด์ (Field) | คำอธิบาย (Description) | บังคับ |
|---|---|---|
| วันที่รับเข้า (Date Received) | วันที่รับวัตถุดิบเข้าคลัง | ✔ |
| ชื่อผู้ส่ง (Sender Name) | ชื่อคนสวน/ซัพพลายเออร์ที่ส่งของ | ✔ |
| รหัสสินค้า (Product Code) | เลือกจากรายการที่ตั้งค่าไว้ | ✔ |
| ประเภทสินค้า (Product Type) | ดึงอัตโนมัติจากรหัสสินค้า | ✔ |
| จำนวน (Quantity, KG) | น้ำหนักที่รับเข้า | ✔ |
| ผู้บันทึก (Recorded By) | ชื่อพนักงานที่บันทึก (ดึงจากผู้ใช้ที่ล็อกอิน/กะวันนั้น) | ✔ (อัตโนมัติ) |
| กะ (Shift) | เช้า/บ่าย/ดึก หรือ ผู้รับผิดชอบวันนั้น | ✔ |
| หมายเหตุ (Note) | ข้อมูลเพิ่มเติม | ✗ |

- รองรับการพูดบันทึกทั้งประโยค เช่น
  *"รับเข้าจาก ยุทธนา วันที่ 10 ตุลาคม ฝา 2 สี หนึ่งร้อยกิโล"*
  ระบบแยกข้อมูลเป็นฟิลด์อัตโนมัติ แล้วให้พนักงานยืนยัน (confirm) ก่อนบันทึกจริง

### 4.3 บันทึกนำออกมาผลิต (Stock Out / Production Issue)
- ระบุ ประเภทสินค้า, จำนวนกิโลกรัมที่นำออก, วันที่, ผู้บันทึก
- ระบบตรวจสอบไม่ให้นำออกเกินยอดคงเหลือ (validation) พร้อมแจ้งเตือน

### 4.4 คำนวณยอดอัตโนมัติ (Auto Calculation)
สำหรับแต่ละประเภทสินค้า ระบบคำนวณและแสดงผลแบบเรียลไทม์:
```
ยอดรวมทั้งหมด (Total In)      = ผลรวมของรายการรับเข้าทั้งหมด
เอาออกมาผลิต (Total Out)       = ผลรวมของรายการนำออกทั้งหมด
คงเหลือ (Remaining Balance)   = ยอดรวมทั้งหมด − เอาออกมาผลิต
```

### 4.5 บันทึกด้วยเสียง (Voice Data Entry)
- ปุ่มไมโครโฟนขนาดใหญ่ กดค้างเพื่อพูด (push-to-talk) หรือแตะเพื่อเริ่ม/หยุด
- แปลงเสียงเป็นข้อความ (Speech-to-Text) รองรับภาษาไทยเป็นหลัก, อังกฤษเป็นทางเลือก
- ระบบแยกคำพูดเป็นฟิลด์ (Name / Date / Product Type / Quantity) โดยใช้การประมวลผลภาษาธรรมชาติ (NLU) แบบกำหนดรูปแบบคำสั่งไว้ล่วงหน้า (intent + slot filling)
- แสดงข้อความที่แปลงได้บนหน้าจอให้ตรวจสอบ พร้อมปุ่ม "ยืนยัน / แก้ไข / พูดใหม่"
- รองรับกรณีเสียงไม่ชัด: ถามกลับเฉพาะฟิลด์ที่ไม่แน่ใจ เช่น "ได้ยินชื่อผู้ส่งว่า 'ยุทธนา' ใช่ไหม?"

### 4.6 รองรับสองภาษา (Bilingual TH/EN)
- สลับภาษา UI ได้จากหน้าตั้งค่า (Settings)
- ชื่อประเภทสินค้าจัดเก็บเป็นคู่ TH/EN เสมอ (เช่น ฝาแดง / Red Cap)
- รายงาน/ใบสรุปสามารถพิมพ์ออกได้ทั้งสองภาษา

### 4.7 การจัดการผู้ใช้และกะทำงาน (User & Shift Management)
- ผู้ดูแลระบบเพิ่ม/ลบพนักงาน กำหนดบทบาท (Recorder / Supervisor / Manager / Admin)
- ตารางกะรายวัน: กำหนดล่วงหน้าว่าวันไหนใครเป็นผู้รับผิดชอบจดบันทึก
- ทุกรายการเก็บ log: ใครบันทึก, เวลาบันทึก, แก้ไขล่าสุดโดยใคร (audit trail)

### 4.8 แดชบอร์ดและรายงาน (Dashboard & Reports)
- ตารางสรุปแบบในภาพต้นฉบับ: รับเข้า, ยอดรวม, นำออกผลิต, คงเหลือ แยกตามประเภทสินค้า
- กรองตามช่วงวันที่, ผู้ส่ง, ประเภทสินค้า, ผู้บันทึก
- Export เป็น Excel/PDF
- แจ้งเตือนเมื่อยอดคงเหลือต่ำกว่าค่าที่กำหนด (low-stock alert) — ฟีเจอร์อนาคต

## 5. โครงสร้างข้อมูล / Data Model (สรุป)

**ProductType**
- product_code (string, unique)
- name_th, name_en (string)
- unit (default: "KG")
- active (boolean)

**StockInEntry**
- id
- date_received
- sender_name
- product_code (FK)
- quantity_kg
- recorded_by (FK: User)
- shift
- note
- created_at, updated_at

**StockOutEntry**
- id
- date_issued
- product_code (FK)
- quantity_kg
- recorded_by (FK: User)
- purpose/note
- created_at, updated_at

**User**
- id, name, role (recorder/supervisor/manager/admin)
- preferred_language (th/en)

**ShiftSchedule**
- date, user_id, shift

## 7. ตัวอย่างการไหลของงาน / Sample User Flow (Voice Entry)

1. พนักงานเปิดแอป → กดปุ่ม "รับเข้าสินค้า" (Stock In)
2. กดปุ่มไมโครโฟน พูด: *"รับเข้าจากไร่อ้อย วันที่ 30 กันยายน ฝาขาวแดง หนึ่งพันห้าร้อยสี่สิบกิโล"*
3. ระบบแปลงเสียง → แสดงฟอร์มที่กรอกอัตโนมัติ:
   - ผู้ส่ง: ไร่อ้อย
   - วันที่: 30/09
   - ประเภท: ฝาขาวแดง
   - จำนวน: 1,540 KG
4. พนักงานตรวจสอบ → กด "ยืนยัน" (หรือแก้ไขด้วยเสียง/พิมพ์ก่อนยืนยัน)
5. ระบบบันทึก พร้อมแนบชื่อผู้บันทึก + เวลา + กะ
6. ยอดรวม/ยอดคงเหลือของ "ฝาขาวแดง" อัปเดตทันทีในแดชบอร์ด

## 8. ขอบเขตที่ยังไม่รวมในเวอร์ชันแรก / Out of Scope (v1)

- การเชื่อมต่อระบบบัญชี/ERP ภายนอก
- การพยากรณ์สต็อกล่วงหน้า (forecasting)

- แจ้งเตือนผ่าน Line/SMS อัตโนมัติ (พิจารณาในเวอร์ชันถัดไป)

## 9. ตัวชี้วัดความสำเร็จ / Success Metrics

- ลดเวลาการบันทึกต่อรายการลง ≥ 50% เทียบกับพิมพ์มือ/กระดาษ
- ความแม่นยำของการแปลงเสียงเป็นข้อมูล ≥ 90% (หลังการยืนยันของผู้ใช้)
- ลดข้อผิดพลาดในการคำนวณยอดคงเหลือเหลือ 0% (เพราะคำนวณอัตโนมัติ)
- พนักงานทุกคนสามารถใช้งานได้โดยไม่ต้องอบรมเกิน 15 นาที
